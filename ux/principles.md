# ShotSpot UX Principles

## Core principles

- Reduce friction for walk-up customers
- Prefer clarity over cleverness
- Make photographer actions explicit and fast
- Keep critical-path decisions visible, not hidden in chat memory
- Write accepted UX decisions into shared docs

## Operating rule

UX ideas are not durable until they are written into:
- `project/decisions.md`
- `project/mvp.md`
- relevant `ux/flows/*.md` docs
